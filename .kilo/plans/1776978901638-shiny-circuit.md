# Plano: Implementação de Compilação de `br` e `cbr` para slynx-to-js

## 1. Visão Geral
Este plano detalha a implementação da compilação das instruções `br` (branch incondicional) e `cbr` (branch condicional) da IR do slynx para JavaScript, incluindo documentação completa para uso como base na codebase.

**Modelo da IR**: Baseado no Cranelift - labels são parametrizadas, eliminando φ-functions. Branches passam argumentos diretamente para os labels alvo.

## 2. Contexto Atual
### 2.1 Estado da Codebase
- **Projeto**: Compilador Rust de IR slynx para JavaScript
- **Arquivos chave**:
  - `src/compiler/mod.rs`: Compilação de contextos/funções
  - `src/js/mod.rs`: Trait `InstructionCompiler` e método `compile_instruction`
  - `src/js/functions.rs`: Struct `JSFunction`
- **Status atual de `Br`/`Cbr`**: Implementações vazias (retornam `""`) em `src/js/mod.rs:187-193`

### 2.2 Estrutura da IR (Modelo Cranelift)
Exemplo de compilação de if-else para IR:
```
entry:
  cbr a, then, else       // branch condicional na condição 'a'
then:
  br final(5)             // branch incondicional para final passando argumento 5
else:
  br final(12)            // branch incondicional para final passando argumento 12
final(int):                // label 'final' com parâmetro 'int'
  ...                     // usa 'int' como valor vindo do branch
```

Conceitos-chave da IR (Cranelift-style):
- **Labels parametrizados**: `final(int)` define um label com parâmetro `int`
- **br**: `br final(5)` - branch incondicional passando argumentos para o label alvo
- **cbr**: `cbr cond, then, else` - branch condicional sem argumentos (argumentos estão nos br subsequentes)
- **Sem φ-functions**: O SSA é implementado através de parâmetros de label, não φ-functions
- **Como funciona**: Quando `br final(5)` executa, o parâmetro `int` no label `final` recebe o valor 5

### 2.3 Desafio
JavaScript não possui `goto` nem labels parametrizados, então é necessário:
1. Reconstruir o fluxo de controle estruturado (if-else, loops) a partir do CFG
2. Converter parâmetros de label em atribuições de variáveis JavaScript
3. Inliner blocos apropriadamente para criar código JavaScript legível

## 3. Estratégia de Implementação
### 3.1 Abordagem Core
1. **Construir CFG**: Mapear cada label para suas instruções, sucessores e predecessores
2. **Rastrear parâmetros de labels**: Armazenar quais parâmetros cada label define
3. **Rastrear argumentos de branches**: Armazenar quais argumentos cada branch passa
4. **Reconhecimento de Padrões**: Identificar if-else (cbr + dois br para mesmo alvo)
5. **Compilar Padrões**: Converter para if-else JS, tratando parâmetros como variáveis

use slynx::middleend::{Label, Instruction, SlynxIR, InstructionType, Value, IRType};
use std::collections::{HashMap, HashSet};

pub struct CFGBlock {
    pub label: Label,
    pub instructions: Vec<Instruction>,
    pub successors: Vec<Label>,           // Labels para os quais este bloco faz branch
    pub predecessors: Vec<Label>,          // Labels que fazem branch para este bloco
    pub params: Vec<(String, IRType)>,    // Parâmetros do label (ex: [("int", IntType)])
    pub branch_args: HashMap<Label, Vec<Value>>, // Argumentos passados para cada sucessor
}

pub struct CFG {
    blocks: HashMap<Label, CFGBlock>, // Label -> CFGBlock
    entry: Label,                     // Bloco de entrada do contexto
}

impl CFG {
    /// Constrói CFG a partir de um contexto da IR slynx
    pub fn from_context(ctx: &slynx::middleend::Context, ir: &SlynxIR) -> Self {
        // 1. Coletar todos os labels no contexto
        let labels: Vec<Label> = ir.get_context_labels(ctx).collect();
        let entry = labels[0].clone(); // Primeiro label é a entrada
        
        // 2. Para cada label, coletar instruções e rastrear sucessores/predecessores
        let mut blocks = HashMap::new();
        let mut label_order: Vec<Label> = Vec::new();
        
        for lbl in &labels {
            label_order.push(lbl.clone());
            let instructions = ir.get_label_instructions(lbl).into_iter().flatten().collect();
            
            // Extrair parâmetros do label a partir da IR
            // TODO: Verificar como a IR do slynx armazena parâmetros de labels
            // (pode estar no próprio Label ou em uma tabela separada)
            let params = Self::extract_label_params(lbl, ir);
            
            let mut block = CFGBlock {
                label: lbl.clone(),
                instructions,
                successors: Vec::new(),
                predecessors: Vec::new(),
                params,
                branch_args: HashMap::new(),
            };
            blocks.insert(lbl.clone(), block);
        }
        
        // 3. Analisar branches para preencher sucessores/predecessores e argumentos
        for lbl in &label_order {
            let block = blocks.get_mut(lbl).unwrap();
            for inst in &block.instructions {
                match &inst.instruction_type {
                    InstructionType::Br(target) => {
                        block.successors.push(target.clone());
                        // Extrair argumentos do branch (estão em inst.operands para br)
                        let args = ir.get_values_by_pointer(inst.operands.clone()).to_vec();
                        block.branch_args.insert(target.clone(), args);
                    }
                    InstructionType::Cbr { then_label, else_label, .. } => {
                        block.successors.push(then_label.clone());
                        block.successors.push(else_label.clone());
                        // cbr não passa argumentos diretamente no Cranelift-style
                        // Os argumentos são passados pelos br subsequentes
                    }
                    _ => {}
                }
            }
        }
        
        // 4. Preencher predecessores
        for (lbl, block) in &mut blocks {
            for succ in &block.successors {
                if let Some(succ_block) = blocks.get_mut(succ) {
                    succ_block.predecessors.push(lbl.clone());
                }
            }
        }
        
        CFG { blocks, entry }
    }

```rust
// Modificações em src/compiler/mod.rs
pub fn compile_context(&mut self, ctx: &Context, ir: &SlynxIR) {
    let name = ctx.name();
    let name = ir.string_pool().get_name(name);
    let types = ir.ir_types();
    let IRType::Function(ty) = types.get_type(ctx.ty()) else { unreachable!() };
    let ty = types.get_function_type(ty);

    let mut func = self.buffer.create_function(name, ty.get_args().len() as u8);
    
    // Construir CFG para este contexto
    let cfg = CFG::from_context(ctx, ir);
    
    // Rastrear blocos já compilados para evitar duplicatas
    let mut compiled_blocks = HashSet::new();
    
    // Iniciar travessia a partir do bloco de entrada
    self.compile_cfg_block(&cfg.entry, &cfg, ir, &mut func, &mut compiled_blocks);
    
    self.buffer.append_function(func);
}

/// Compila um bloco do CFG e seus sucessores recursivamente
fn compile_cfg_block(
    &mut self,
    block_label: &Label,
    cfg: &CFG,
    ir: &SlynxIR,
    func: &mut JSFunction,
    compiled_blocks: &mut HashSet<Label>,
) {
    if compiled_blocks.contains(block_label) {
        return;
    }
    compiled_blocks.insert(block_label.clone());
    
    let block = cfg.blocks.get(block_label).unwrap();
    
    // Compilar todas as instruções deste bloco exceto a branch final
    let branch_inst = block.instructions.last();
    for inst in &block.instructions[..block.instructions.len().saturating_sub(1)] {
        let result = func.compile_instruction(inst, ir);
        if !result.is_empty() {
            func.append(result);
        }
    }
    
    // Tratar a instrução de branch final (se houver)
    if let Some(branch_inst) = branch_inst {
        match &branch_inst.instruction_type {
            InstructionType::Br(target) => {
                self.handle_unconditional_branch(target, block_label, cfg, ir, func, compiled_blocks);
            }
            InstructionType::Cbr { then_label, else_label, .. } => {
                self.handle_conditional_branch(
                    then_label, else_label,
                    branch_inst, cfg, ir, func, compiled_blocks
                );
            }
            _ => {
                // Última instrução não é branch, compilar normalmente
                let result = func.compile_instruction(branch_inst, ir);
                if !result.is_empty() {
                    func.append(result);
                }
            }
        }
    }
}
```

### Passo 3: Implementar Compilação de Branch Condicional (cbr)
Tratar `cbr` detectando padrões de if-else. No modelo Cranelift, os branches `then` e `else` terminam com `br` que passa argumentos para o bloco de merge.

```rust
// Adições em src/compiler/mod.rs
fn handle_conditional_branch(
    &mut self,
    then_label: &Label,
    else_label: &Label,
    branch_inst: &Instruction,
    cfg: &CFG,
    ir: &SlynxIR,
    func: &mut JSFunction,
    compiled_blocks: &mut HashSet<Label>,
) {
    // Compilar condição
    let condition = func.compile_values(
        ir.get_values_by_pointer(branch_inst.operands.clone()),
        ir
    ).join(", ");
    
    // Verificar se ambos os branches vão para o mesmo bloco de merge (padrão if-else)
    let then_block = cfg.blocks.get(then_label).unwrap();
    let else_block = cfg.blocks.get(else_label).unwrap();
    
    // Pegar o alvo do br final de cada bloco
    let then_target = then_block.successors.first().cloned();
    let else_target = else_block.successors.first().cloned();
    
    if then_target == else_target && then_target.is_some() {
        // Padrão if-else: ambos then/else fazem branch para o mesmo bloco de merge
        let merge_label = then_target.unwrap();
        let merge_block = cfg.blocks.get(&merge_label).unwrap();
        
        // Extrair argumentos dos branches (br) dentro dos blocos then e else
        let then_args = then_block.branch_args.get(&merge_label).cloned().unwrap_or_default();
        let else_args = else_block.branch_args.get(&merge_label).cloned().unwrap_or_default();
        
        // Iniciar bloco if-else
        func.append(format!("if ({}) {{\n", condition));
        
        // No bloco then: atribuir argumentos às variáveis
        for (i, arg) in then_args.iter().enumerate() {
            let arg_str = func.compile_values(&[arg.clone()], ir).join(", ");
            func.append(format!("let v{} = {};\n", i + 1, arg_str));
        }
        // Compilar instruções do bloco then (exceto o br final)
        self.compile_block_content(then_label, cfg, ir, func, false);
        func.append("} else {\n".to_string());
        
        // No bloco else: atribuir argumentos às variáveis
        for (i, arg) in else_args.iter().enumerate() {
            let arg_str = func.compile_values(&[arg.clone()], ir).join(", ");
            func.append(format!("let v{} = {};\n", i + 1, arg_str));
        }
        // Compilar instruções do bloco else (exceto o br final)
        self.compile_block_content(else_label, cfg, ir, func, false);
        func.append("}\n".to_string());
        
        // Continuar compilação a partir do bloco de merge
        self.compile_cfg_block(&merge_label, cfg, ir, func, compiled_blocks);
    } else {
        // Fluxo de controle não estruturado: emitir if-else JS sem merge compartilhado
        func.append(format!("if ({}) {{\n", condition));
        self.compile_cfg_block(then_label, cfg, ir, func, compiled_blocks);
        func.append("} else {\n".to_string());
        self.compile_cfg_block(else_label, cfg, ir, func, compiled_blocks);
        func.append("}\n".to_string());
    }
}

/// Compila conteúdo do bloco excluindo a instrução de branch final
fn compile_block_content(
    &self,
    block_label: &Label,
    cfg: &CFG,
    ir: &SlynxIR,
    func: &mut JSFunction,
    include_branch: bool,
) {
    let block = cfg.blocks.get(block_label).unwrap();
    let end = if include_branch { block.instructions.len() } else { block.instructions.len().saturating_sub(1) };
    
    for inst in &block.instructions[..end] {
        let result = func.compile_instruction(inst, ir);
        if !result.is_empty() {
            func.append(result);
        }
    }
}
```

### Passo 4: Implementar Compilação de Branch Incondicional (br)
Tratar `br` passando argumentos para o label alvo. No modelo Cranelift, `br final(5)` faz o controle passar para `final` com parâmetro recebendo 5.

```rust
// Adições em src/compiler/mod.rs
fn handle_unconditional_branch(
    &mut self,
    target_label: &Label,
    source_label: &Label,
    cfg: &CFG,
    ir: &SlynxIR,
    func: &mut JSFunction,
    compiled_blocks: &mut HashSet<Label>,
) {
    let target_block = cfg.blocks.get(target_label).unwrap();
    let source_block = cfg.blocks.get(source_label).unwrap();
    
    // Obter argumentos que este branch passa para o alvo
    let args = source_block.branch_args.get(target_label).cloned().unwrap_or_default();
    
    // Se o alvo tem parâmetros, atribuir os argumentos às variáveis correspondentes
    if !target_block.params.is_empty() && !args.is_empty() {
        for (i, (_, _)) in target_block.params.iter().enumerate() {
            if i < args.len() {
                let arg_str = func.compile_values(&[args[i].clone()], ir).join(", ");
                let var_name = format!("v{}", func.variables().len() + 1 + i);
                func.append(format!("let {} = {};\n", var_name, arg_str));
            }
        }
    }
    
    // Verificar se o alvo é um bloco de retorno (última instrução é Ret)
    if let Some(last_inst) = target_block.instructions.last() {
        if matches!(last_inst.instruction_type, InstructionType::Ret) {
            // Compilar o bloco alvo até o retorno
            self.compile_block_content(target_label, cfg, ir, func, true);
            return;
        }
    }
    
    // Caso contrário, continuar compilação no bloco alvo
    self.compile_cfg_block(target_label, cfg, ir, func, compiled_blocks);
}
```

## 5. Notas de Documentação
### Por que Abordagem Baseada em CFG?
- A IR do slynx usa um CFG explícito com blocos básicos e branches (modelo Cranelift)
- JavaScript requer fluxo de controle estruturado, então é necessário reconstruí-lo a partir do CFG
- O reconhecimento de padrões (if-else, loops) permite compilação correta dos casos comuns
- Labels parametrizados eliminam necessidade de φ-functions

### Como Funciona o Modelo Cranelift na IR slynx
1. **Labels parametrizados**: `final(int)` define um label que aceita um parâmetro `int`
2. **Branches passam argumentos**: `br final(5)` faz o controle ir para `final` com `int = 5`
3. **Sem φ-functions**: O SSA é mantido através dos parâmetros de label, não φ-functions
4. **Exemplo de if-else**:
   - `cbr cond, then, else` - decide qual bloco executar
   - `then: br final(5)` e `else: br final(12)` - ambos vão para `final` com valores diferentes
   - `final(int):` - recebe o valor 5 ou 12 baseado em qual branch veio

### Premissas
1. Instruções `br` passam argumentos armazenados em `instruction.operands`
2. `cbr` não passa argumentos diretamente; os argumentos estão nos `br` subsequentes
3. Labels podem ter parâmetros (extrair da IR do slynx)
4. Todos os labels são alcançáveis a partir do bloco de entrada
5. Fluxo de controle aninhado pode ser tratado recursivamente

### Limitações da Implementação Inicial
- Apenas trata padrões if-else simples (ainda não loops)
- Tratamento de parâmetros de labels é simplificado (assume tipos conhecidos)
- Fluxo de controle não estruturado (branches não correspondentes) pode não compilar corretamente
- Não trata loops (será implementado em versão futura)
